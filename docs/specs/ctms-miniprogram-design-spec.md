# CTMS 小程序产品设计规范 (PRD/Spec)

## 1. 概述 (Overview)

**背景：** 当前 CTMS 系统 Web 端已基本完成，小程序端目前仅包含 IWRS 逻辑。随着 EDC 电子数据采集系统的并入，需要重新设计小程序架构，使其能够动态兼容 IWRS 和 EDC 模块，同时保证良好、连贯的用户体验。

**目标：** 设计一套模块化、基于角色的 CTMS 小程序架构。不通过底部 Tab 生硬拆分系统，而是以「项目」和「角色」为核心进行降维融合，确保系统具备高度的可扩展性与数据库友好性。

## 2. 核心产品原则 (Core Principles)

- **底座架构统一：** 底部导航栏（Tab Bar）固定为「工作台」、「项目」、「我的」三项。坚决不以系统模块（IWRS/EDC）作为一级导航划分。
- **多角色独立视图 (RBAC)：** 根据登录角色的不同（如 CRC、医生/PI），提供高度定制化的首页工作台视图。
- **项目维度降维 (Project-Centric)：** 将 IWRS 和 EDC 的系统层级降维到「项目」内。同一个项目下，通过「文件夹式 Tab」隔离不同系统的状态与数据，解决业务阶段重叠带来的状态冲突。
- **数据库友好的卡片流：** UI 展示与底层数据字段严格对应，枚举值标准化，时间戳独立分离，避免前端强耦合业务文本。

## 3. 角色与工作台设计 (Role & Workbench Design)

### 3.1 底部导航 (Bottom Navigation)

- **工作台 (Workbench)：** 核心行动区，处理待办、录入数据、快捷操作。
- **项目 (Projects)：** 项目列表，查看各项目宏观进度与模块详情。
- **我的 (Mine)：** 个人中心、系统设置、消息通知。

### 3.2 CRC 视角 (CRC View)

- **核心诉求：** 任务执行与待办处理（如录入数据、处理系统质疑、受试者随访跟进）。
- **页面布局：**
  - **顶部模块：** 待办事项（To-do Items）总览。
  - **卡片流：** 按时间倒序排列的待办卡片，取代时间轴设计，避免拥挤。
  - **颜色规范：** IWRS 相关任务严格使用蓝色（Blue），EDC 相关任务严格使用绿色（Green）。

### 3.3 医生/研究者视角 (Doctor/PI View)

- **核心诉求：** 宏观审核、异常关注、便捷入组预约。
- **页面布局：**
  - **顶部模块：** 包含便捷入口（如预约入组、受试者查询）。
  - **数据看板：** 重点关注的核心指标（如异常数据提醒、严重不良事件 SAE）。
  - **卡片流：** 需要医生审核或签名的待办卡片。

## 4. 核心页面设计 (Core Page Design)

### 4.1 首页工作台 (Home / Workbench)

- **卡片设计规范：**
  - **标签体系：** 左侧标识系统类型（如「系统质疑」、「待录入」），右上角标识优先级枚举（高/中/低优先级）。
  - **时间字段：** 独立时间展示。如果是今天的时间，显示格式为 `今天 HH:mm`；如果是历史时间，显示为 `MM-DD HH:mm`。
  - **色彩体系：** 卡片背景、边框、标签颜色严格遵守模块映射（IWRS = 蓝色，EDC = 绿色）。

### 4.2 项目列表页 (Project List)

- **文件夹式 Tab 设计：**
  - 在每个项目卡片内部使用文件夹风格的 Tab 切换状态展示。
  - 选中状态的 Tab 紧贴容器边缘（无缝衔接），未选中状态置灰。
  - **自适应逻辑：** 如果项目仅购买单系统（如仅 IWRS），则直接显示单系统信息，无内部 Tab 切换。
- **宏观状态标签：** 项目整体阶段分为三种枚举，并绑定固定颜色：
  - `筹备期`：蓝色
  - `进行中`：绿色
  - `已结束`：灰色

### 4.3 项目详情页 (Project Detail)

- **顶部宏观卡片：** 置顶展示跨系统的基础项目信息（如项目编号、申办方、主要研究者、阶段）。若为双系统项目，顶部可增加渐变色条作为视觉暗示。
- **胶囊文件夹式 Tab 切换：**
  - 紧贴下方的内容容器，实现无缝融合。
  - **IWRS Tab：** 展示入组进度、受试者漏斗数据看板、新增受试者/分配药物等快捷操作、近期受试者动态列表。
  - **EDC Tab：** 展示 CRF 整体完成度、核心数据看板（带红点提醒的未解决质疑、SDV 完成率）、录入数据/处理质疑等快捷操作、待办录入列表。
- **单模块自适应：** 后端接口返回单系统权限时，隐藏模块 Tab 栏，内容容器直接铺满单系统专属内容。

## 5. 数据结构与字段映射 (Data Structure Mapping)

为保证前端 UI 与后端数据库的解耦与高效映射，定义以下核心字段规范：

### 5.1 任务/待办表 (Task / Todo)

| 字段名 (Field) | 类型 (Type) | 必填 (Required) | 说明 (Description) |
|----------------|-------------|-----------------|--------------------|
| `module_type` | enum | 是 | 所属模块：`IWRS` (前端渲染蓝色) / `EDC` (前端渲染绿色) |
| `priority` | enum | 是 | 优先级：`HIGH` (高) / `MEDIUM` (中) / `LOW` (低) |
| `task_type` | enum | 是 | 任务类型：`ENTRY` (待录入) / `QUERY` (质疑) / `SIGN` (待签名) 等 |
| `reminder_time` | datetime | 是 | 提醒/触发时间，前端根据此字段格式化显示日期时间 |

### 5.2 项目与模块状态映射 (Project & Module Status)

| 字段名 (Field) | 类型 (Type) | 必填 (Required) | 说明 (Description) |
|----------------|-------------|-----------------|--------------------|
| `macro_status` | enum | 是 | 项目宏观状态：`PREP` (筹备期) / `IN_PROGRESS` (进行中) / `ENDED` (已结束) |
| `purchased_modules` | array[enum] | 是 | 已购模块列表，例如：`['IWRS', 'EDC']`，前端据此决定是否渲染文件夹 Tab |
| `iwrs_status` | json | 否 | IWRS 模块相关进度数据（入组数、漏斗数据等） |
| `edc_status` | json | 否 | EDC 模块相关进度数据（完成度、质疑数等） |

**文档说明：** 
本文档为 CTMS 小程序端 UI/UX 与核心逻辑的指导性规范。后续实际的 React (Taro/Uniapp) 组件开发均应以本规范的原则与字段约束为准。
